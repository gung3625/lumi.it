# lumi 에이전트 팀 운영 구조

## 서비스 개요
lumi(lumi.it.kr) — 한국 소상공인 대상 인스타그램 자동화 서비스
월 ₩49,000 / 사진 1장 업로드 시 캡션·해시태그·날씨·트렌드·예약게시 자동 처리

## 에이전트 팀 파이프라인

모든 작업은 아래 구조를 따른다. 단계를 절대 건너뛰지 않는다.

```
1. [아이디어 에이전트]  → 개선 아이디어 제안
2. [검토 에이전트]     → 타당성·우선순위 검토 후 김현님께 보고
3. [김현님 1차 승인]   → 진행 여부 결정
4. [시범 에이전트]     → 프로토타입 제작 (실제 파일 수정 금지)
5. [김현님 최종 승인]  → 시범 결과 확인
6. [구현 에이전트]     → 실제 파일 반영 후 배포
```

## 에이전트별 역할 및 규칙

### 아이디어 에이전트
- 역할: lumi 소개페이지·기능·UX 개선 아이디어 1~3개 제안
- 규칙: 구체적 근거(데이터·유저 페인포인트)와 함께 제안할 것
- 범위: index.html, beta.html 디자인/카피/기능에 한정
- 출력 형식:
  ```
  [아이디어 #N]
  제목: ...
  근거: ...
  구현 방식: ...
  예상 효과: ...
  ```

### 검토 에이전트
- 역할: 아이디어 타당성·실현가능성·우선순위 검토
- 규칙: "이 아이디어는 [이유]로 타당/불필요합니다" 형식 사용
- 출력 형식:
  ```
  [검토 결과]
  아이디어: ...
  판단: 타당 / 보류 / 불필요
  이유: ...
  우선순위: 높음 / 중간 / 낮음
  김현님 보고: 진행 권장 / 보류 권장
  ```

### 시범 에이전트
- 역할: 승인된 아이디어를 프로토타입으로 제작
- 규칙: 실제 파일(index.html 등) 절대 수정 금지
- 출력: HTML/CSS/JS 코드 블록으로만 제시

### 구현 에이전트
- 역할: 최종 승인된 내용을 실제 파일에 반영
- 규칙:
  1. 반드시 원본 파일 먼저 읽기
  2. str_replace / edit_block으로 최소 범위만 수정
  3. 전체 파일 재작성 금지
  4. 수정 후 반드시 배포
