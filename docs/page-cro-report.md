# lumi 랜딩페이지 CRO 분석 리포트

**분석 대상**: https://lumi.it.kr (index.html)
**페이지 유형**: Homepage (프리런칭 베타)
**전환 목표**: 무료 베타 테스터 신청 (/beta)
**분석일**: 2026-04-09

---

## 1. Value Proposition Clarity (가치 제안 명확성)

**현재 상태**: 중상

**잘 된 점**:
- "사진만 올리면 AI가 캡션 써주고 인스타에 올려줌"이라는 핵심 가치가 명확함
- 타겟(소상공인)의 언어로 작성됨 — 기술 용어 없음
- Before/After 섹션이 가치를 직관적으로 보여줌

**문제점**:
- 헤드라인 "우리 매장을 아는 lumi가 대신 써드려요"는 **lumi가 뭔지 모르는 첫 방문자에겐 모호함**. "대신 써드려요"만으로는 뭘 써주는지 불분명
- 히어로 서브카피에서 "날씨, 동네 행사, 우리 매장 말투까지 이해하고"는 **차별점이지만 첫 5초에 전달하기엔 정보 과부하**
- "lumi"라는 브랜드명만으로는 서비스 카테고리를 유추하기 어려움

**점수**: 7/10

## 2. Headline Effectiveness (헤드라인 효과)

**현재 헤드라인**: "우리 매장을 아는 lumi가 대신 써드려요."
**현재 서브**: "날씨, 동네 행사, 우리 매장 말투까지 이해하고 사진 한 장으로 캡션 · 해시태그 · 게시까지 전부 자동."

**분석**:
- "대신 써드려요"는 감성적이지만, **구체적 결과(outcome)**가 빠져 있음
- 경쟁사 대비 차별점(날씨/행사 반영)이 서브카피에 묻혀 있음
- 숫자/구체성 부족 — "37분 → 1분"이 OG 태그에만 있고 히어로에는 없음
- hero-chip "베타 테스터 모집 중"은 좋지만, 긴급성(urgency) 부족

**점수**: 6/10

## 3. CTA Placement, Copy, and Hierarchy (CTA 배치·카피·위계)

**현재 CTA 구조**:
| 위치 | CTA 텍스트 | 타입 |
|------|-----------|------|
| 배너 | "신청" | 보조 |
| 네비 | "무료 테스터 신청" | 보조 |
| 히어로 | "무료 테스터 신청" + "직접 체험하기" | **Primary + Secondary** |
| 데모 결과 후 | "무료 테스터로 시작하기" | Primary |
| 가격표 | "시작하기" / "무료 테스터 신청하기" | Primary |
| 최하단 CTA | "무료 테스터 신청하기" | Primary |
| Sticky (모바일) | "무료 테스터 신청하기" | Primary |

**잘 된 점**:
- Primary/Secondary CTA 구분이 명확함 (핑크 filled vs outline)
- 스크롤 내려도 반복적으로 CTA 노출
- 모바일 Sticky CTA 존재
- "직접 체험하기" 세컨더리 CTA로 '아직 준비 안 된' 방문자 포착

**문제점**:
- **CTA 카피가 전부 동일** ("무료 테스터 신청") — 컨텍스트별 차별화 없음
- 가격표 베이직/프로의 "시작하기"는 행동이 불명확 (시작이 뭔지?)
- "무료 테스터 신청"이 **가치를 전달하지 않음** — "무료로 시작하기"보다 약함
- CTA 최하단 "잔여 6자리"가 **하드코딩**되어 실제 잔여와 불일치 가능

**점수**: 7/10

## 4. Visual Hierarchy and Scannability (시각적 위계·스캔성)

**잘 된 점**:
- Apple 디자인 시스템 기반으로 깔끔한 시각 위계
- 다크/라이트 섹션 교대로 시각적 리듬감
- 섹션 라벨(sec-label) → 타이틀 → 콘텐츠 구조가 일관됨
- HOW 섹션의 인터랙티브 폰 모형이 몰입감 높음
- reveal 애니메이션이 적절한 수준

**문제점**:
- **히어로 폰 목업이 너무 작음** (240x480px) — 캡션 텍스트가 9px로 거의 안 읽힘
- Features 섹션 카드가 **12칼럼 그리드 7:5 / 5:7** 배치인데, 4개 카드 중 어느 것이 핵심인지 위계가 불분명
- "모든 기능 보기" 버튼 뒤에 숨겨진 기능이 8개나 됨 — **핵심 기능이 가려질 수 있음**
- Metrics 스트립의 숫자들이 히어로 바로 아래에 있어 히어로의 임팩트를 분산시킴
- 후기 카드 4개가 다크모드에서 **흰색 배경 하드코딩**(`background:#fff`)으로 인해 다크모드에서 이질감

**점수**: 7/10

## 5. Trust Signals and Social Proof (신뢰 시그널)

**현재 신뢰 요소**:
- 4개 고객 후기 (이름 + 업종 + 별점 + 구체적 경험담)
- Before/After 비교
- Metrics (평균 47초, 4.9점 만족도)
- 사업자 등록번호, 통신판매업 신고번호 (Footer)
- FAQ에서 데이터 보안, Meta 공식 API 언급

**문제점**:
- **테스터 0명인데 후기 4개** — 실제 사용자 후기인지 의문이 생길 수 있음. "4.9점 테스터 평균 만족도" 역시 마찬가지
- 후기에 사진이 없음 — 신뢰도 감소
- **로고/브랜드 파트너** 섹션 없음 (Meta/Instagram 공식 파트너 배지 등)
- **미디어 노출** 섹션 없음
- 숫자 "14/20"은 실시간이지만, **만족도 4.9점은 출처 불분명**
- 보안 배지(SSL, Meta 인증 등) 없음

**점수**: 5/10 — **가장 개선 필요한 영역**

## 6. Objection Handling (반론 처리)

**현재 FAQ에서 다루는 반론**:
1. "베타 끝나면 바로 유료?" → 2주 전 안내, 자동 결제 없음
2. "글이 어색하지 않나?" → 말투 학습, 수정 가능
3. "사진이 다른 곳에 쓰이나?" → 외부 학습 안 함
4. "계정 연동 안전한가?" → Meta 공식 API
5. "하루 몇 개?" → 캡션 무제한, 게시 5개

**잘 된 점**:
- FAQ가 실제 소상공인이 물을 법한 질문으로 구성됨
- 답변이 구체적이고 간결함

**누락된 반론**:
- **"대행사보다 나은 점이 뭐야?"** — 가격표에서 간접 비교하지만 명시적 비교 없음
- **"ChatGPT로 직접 하면 안 돼?"** — 핵심 경쟁 시나리오인데 다루지 않음
- **"인스타만 되나? 블로그/네이버는?"** — 범위 불분명
- **"매장이 작아서 인스타까지 할 여유가 없는데"** — 타겟의 근본 반론 미처리
- **가격 앵커링** 부족 — "대행사 월 50~100만원 vs lumi 1.9만원" 비교가 없음

**점수**: 6/10

## 7. Friction Points (마찰 요소)

**발견된 마찰**:
1. **데모 체험이 하루 3회로 제한** — 처음 방문한 사람에게 너무 엄격. 첫 체험은 자유롭게 하고 2회차부터 제한하는 게 나음
2. **데모에서 업종 선택 → 사진 업로드 → 버튼 클릭** 3단계 — 업종 선택을 선택사항으로 만들거나 자동 감지하면 마찰 감소
3. **"무료 테스터 신청" 클릭 시 /beta 페이지로 이동** — 추가 단계가 전환율을 낮출 수 있음 (인라인 신청 폼 고려)
4. **히어로 → 데모 섹션까지 스크롤 거리가 멂** — HOW 섹션(4 스텝)과 FEATURES 섹션을 지나야 데모에 도달
5. **다크모드 기본** — 소상공인 타겟은 다크모드에 익숙하지 않을 수 있음
6. **배너 "14/20" 숫자의 의미**가 직관적이지 않음 — "6자리 남음"이 더 명확
7. **CTA 최하단 "잔여 6자리"가 하드코딩** — 배너와 불일치 시 신뢰 하락
8. **Unsplash 스톡 이미지** 사용 — 실제 매장 사례가 아니라 제품 신뢰도 감소

**점수**: 6/10

## Quick Wins (바로 적용)

### QW-1. 히어로 서브카피에 숫자 추가
**현재**: "날씨, 동네 행사, 우리 매장 말투까지 이해하고..."
**변경**: "매일 37분 → 단 1분. 사진 한 장이면 캡션·해시태그·게시까지 자동."
**이유**: OG 태그에 이미 있는 "37분 → 1분" 데이터를 히어로에 활용. 구체적 숫자가 전환율에 직접 영향.

### QW-2. CTA 최하단 잔여 수를 동적으로
**현재**: "잔여 6자리" 하드코딩
**변경**: 배너와 같이 API에서 실시간 조회 (20 - count)
**이유**: 배너와 CTA 박스의 숫자 불일치는 신뢰를 깨뜨림.

### QW-3. 가격 앵커링 한 줄 추가
**위치**: 가격표 섹션 타이틀 아래
**추가**: "대행사 월 50~100만원 vs lumi 월 1.9만원부터"
**이유**: 가격 저항을 10x 저렴함으로 즉시 해소.

### QW-4. 후기 카드 다크모드 대응
**현재**: `background:#fff` 하드코딩
**변경**: 다크모드에서 `var(--surface-2)` 적용
**이유**: 다크모드에서 후기 카드가 눈에 띄게 이질적.

### QW-5. 데모 섹션 첫 회차 제한 해제
**현재**: 하루 3회 전체 제한
**변경**: 첫 1회는 제한 없이 체험 가능, 2회차부터 3회/일 제한
**이유**: 첫 방문자의 체험 마찰을 줄여 퍼널 진입률 향상.

## High-Impact Changes (우선순위 높은 변경)

### HI-1. 히어로 헤드라인 재구성
현재 헤드라인이 lumi를 모르는 사람에게 모호함. "인스타 글, 사진만 올리면 끝"처럼 결과 중심으로 변경.

**구현 방향**:
- 헤드라인: 결과(Outcome) 중심
- 서브카피: 차별점(날씨, 말투, 행사) 강조
- 구체적 숫자 포함

### HI-2. "ChatGPT vs lumi" 비교 섹션 추가
소상공인의 실제 대안은 ChatGPT 복붙임. Before/After 섹션 근처에 "ChatGPT로 직접 하기 vs lumi" 비교를 추가하면 전환 장벽을 낮출 수 있음.

| 항목 | ChatGPT 복붙 | lumi |
|------|-------------|------|
| 사진 분석 | 직접 설명 입력 | 자동 인식 |
| 해시태그 | 직접 검색 | 실시간 트렌드 |
| 인스타 게시 | 복붙 후 직접 | 원클릭 자동 |
| 소요 시간 | 15~20분 | 1분 |

### HI-3. 데모 섹션을 히어로 바로 아래로 이동
현재 데모까지 HOW(4단계) + FEATURES(4+8카드)를 지나야 함. "직접 체험하기" CTA의 앵커가 #demo인데, 스크롤 거리가 너무 길어 이탈 발생 가능.

**제안**: 섹션 순서 변경
```
히어로 → Metrics → 데모(체험) → HOW(사용법) → Features → Proof → Pricing → CTA → FAQ
```

### HI-4. 실제 매장 스크린샷/사례로 교체
현재 Unsplash 스톡 이미지를 사용 중. 실제 테스터가 생기면 실제 매장 사진 + 실제 생성된 캡션으로 교체. 특히 폰 목업 내 컨텐츠.

### HI-5. 신뢰 시그널 강화
- "Meta 공식 API 사용" 배지를 히어로 또는 CTA 근처에 추가
- "카드 등록 없음 · 자동 결제 없음 · 언제든 해지" 3요소를 CTA 바로 아래에 명시
- 후기에 실제 인스타 게시물 스크린샷 추가 (증거)

## Test Ideas (A/B 테스트 아이디어)

### Test 1: 헤드라인 변형
- **A** (현재): "우리 매장을 아는 lumi가 대신 써드려요."
- **B**: "사진 한 장 올리면, 인스타 글은 lumi가 써요."
- **C**: "인스타 글 쓰는 데 37분? 이제 1분이면 돼요."
- **가설**: 구체적 숫자(C)가 감성 카피(A)보다 전환율이 높을 것

### Test 2: 히어로 CTA 카피
- **A** (현재): "무료 테스터 신청"
- **B**: "무료로 시작하기"
- **C**: "1분 만에 캡션 받아보기"
- **가설**: 행동의 결과를 보여주는 C가 전환율이 높을 것

### Test 3: 데모 → 가입 퍼널 순서
- **A** (현재): 히어로 → HOW → Features → 데모 → Proof → Pricing
- **B**: 히어로 → 데모 → Proof → Features → Pricing
- **가설**: 데모를 먼저 체험시키면 이후 섹션의 설득력이 높아질 것

### Test 4: 데모 결과 후 CTA
- **A** (현재): 결과 + 업그레이드 카드 + 비교표 동시 노출
- **B**: 결과만 보여주고, "이 캡션으로 바로 인스타에 올리고 싶으세요?" → 가입 CTA
- **가설**: 정보 과부하를 줄이면 가입 전환율이 올라갈 것

### Test 5: 다크모드 vs 라이트모드 기본값
- **A** (현재): 다크모드 기본
- **B**: 라이트모드 기본
- **가설**: 40~60대 소상공인 타겟은 라이트모드가 더 익숙하여 이탈률이 낮을 것
