# lumi 베타 신청 플로우 — Signup Flow CRO 리포트

> **분석 대상**: beta.html (https://lumi.it.kr/beta)
> **플로우 유형**: Waitlist / Early Access (선착순 20명 무료 베타 테스터)
> **분석일**: 2026-04-09

---

## 1. 현재 플로우 요약

### 플로우 구조 (Single-Page, Single-Step)
```
[상단 배너 CTA] → 폼으로 스크롤
[Hero CTA 버튼] → 폼으로 스크롤
[Sticky CTA (모바일)] → 폼으로 스크롤
        ↓
[신청 폼: 5개 필드] → POST /api/beta-apply → 완료 화면
```

### 현재 필드 구성
| 순서 | 필드 | 타입 | 필수 여부 | 비고 |
|------|------|------|----------|------|
| 1 | 이름 | text | 필수 | placeholder: 홍길동 |
| 2 | 매장명 | text | 필수 | placeholder: 성수 카페 루미 |
| 3 | 업종 | select | 필수 | 5개 옵션 + 빈 값 |
| 4 | 연락처 | tel | 필수 | placeholder: 010-0000-0000 |
| 5 | 인스타그램 계정 | text | 선택 | placeholder: @우리매장 |

### 페이지 섹션 흐름
배너 → Nav → Hero → How(사용법) → 테스터 혜택 → 기능 소개(11개) → Before/After → **신청 폼** → FAQ → Footer

## 2. Audit Findings

### Issue #1: 폼이 페이지 하단에 위치 — 도달률 저하
- **Issue**: 신청 폼이 Hero → How → 혜택 → 기능 11개 → Before/After 이후에 나옴. 스크롤 깊이가 매우 깊어 폼 도달 전 이탈 가능성 높음
- **Impact**: 높음. 모바일에서 폼까지 최소 8~10회 스와이프 필요. 관심 있어도 폼을 찾지 못하거나 스크롤 피로로 이탈
- **Fix**: Hero 영역에 간소화된 인라인 폼(이름 + 연락처만) 배치하거나, 폼 위치를 Before/After 직전으로 이동
- **Priority**: **High**

### Issue #2: 필수 필드 4개 — 베타 신청치고 과다
- **Issue**: 이름, 매장명, 업종, 연락처 4개 필수. Waitlist/Early Access 패턴에서 필수 필드는 1~2개가 최적 (이메일 또는 연락처 + 이름)
- **Impact**: 높음. 필드 하나 추가될 때마다 전환율 ~5-10% 감소. 매장명/업종은 신청 후 온보딩에서 수집 가능
- **Fix**: 필수를 이름 + 연락처로 축소. 매장명/업종은 선택으로 변경하거나 신청 후 알림톡에서 수집
- **Priority**: **High**

### Issue #3: 인라인 유효성 검사 없음
- **Issue**: 모든 검증이 submit 시점 `alert()`로 처리됨. 어떤 필드가 비었는지 구체적으로 알려주지 않음
- **Impact**: 중간. 사용자가 어디를 놓쳤는지 모르고, alert은 모바일에서 경험이 나쁨
- **Fix**: 각 필드에 인라인 에러 표시 (빨간 테두리 + 하단 메시지). 연락처는 실시간 형식 검증 추가
- **Priority**: **Medium**

### Issue #4: 연락처 입력 UX 부족
- **Issue**: `input type="tel"` 사용하지만, 자동 포맷팅(하이픈 삽입)이나 숫자만 입력 제한이 없음. placeholder만 있음
- **Impact**: 낮음~중간. 잘못된 형식으로 신청 가능. 운영자가 연락 불가한 번호를 받을 수 있음
- **Fix**: 입력 시 자동 하이픈 삽입 (`010-1234-5678`). 숫자 외 문자 필터링. 11자리 검증
- **Priority**: **Medium**

### Issue #5: CTA 버튼 카피 반복 + 폼 내 신뢰 문구 중복
- **Issue**: 폼 하단에 두 줄의 신뢰 문구가 거의 동일한 내용으로 중복됨
  - "언제든 탈퇴 가능 · 카드 등록 없음 · 24시간 내 안내"
  - "신용카드 불필요 · 스팸 없음 · 언제든 취소 가능"
- **Impact**: 낮음. 중복은 전문성을 떨어뜨리고 시각적 노이즈 추가
- **Fix**: 하나로 통합: "카드 등록 없음 · 언제든 탈퇴 가능 · 24시간 내 안내"
- **Priority**: **Low**

### Issue #6: 폼 상단 메시지가 중요 정보를 반복
- **Issue**: 폼 위에 "사진 한 장이면 캡션부터 게시까지 전부 자동" 텍스트가 별도 div로 있고, 그 아래 form-card에 또 "lumi를 먼저 무료로 써보세요" + 슬롯 프로그레스가 있음. Hero에서 이미 전달된 정보
- **Impact**: 낮음. 스크롤 깊이 추가 + 폼 시작까지 거리 증가
- **Fix**: 폼 카드 내부에 핵심 메시지만 유지. 외부 p 태그 제거
- **Priority**: **Low**

### Issue #7: 완료 화면의 다음 단계 불명확
- **Issue**: 신청 완료 후 "카카오 알림톡으로 안내 드릴게요" + 공유 버튼 + "홈으로 가기" 버튼. 구체적으로 언제, 무엇을 안내받는지 불명확
- **Impact**: 중간. 기대 관리 실패 → 불안 → 재문의 또는 이탈
- **Fix**: "24시간 내 카카오톡으로 사용 안내를 보내드려요. 연락처를 확인해 주세요." 처럼 구체적 타임라인 + 액션 명시
- **Priority**: **Medium**

### Issue #8: 업종 select의 옵션이 제한적
- **Issue**: 카페, 식당, 뷰티, 쇼핑, 기타 5개만. 타겟에 꽃집, 베이커리, 헬스장, 인테리어, 반려동물이 포함되는데 모두 "기타"로 분류됨
- **Impact**: 낮음~중간. "기타"를 선택하면 데이터 가치 하락 + 사용자가 "내 업종이 없네" 느낌
- **Fix**: 옵션 추가 (베이커리·디저트, 꽃집·플라워, 헬스·필라테스, 반려동물, 인테리어·소품) 또는 자유 입력 허용
- **Priority**: **Low**

### Issue #9: 소셜 로그인 / 간편 신청 옵션 없음
- **Issue**: 카카오 로그인으로 이름 + 연락처 자동 수집 가능하지만, 수동 입력만 존재
- **Impact**: 높음. 타겟이 한국 소상공인이므로 카카오 간편 신청이 가장 자연스러운 채널. 타이핑 부담 대폭 감소
- **Fix**: "카카오로 간편 신청" 버튼 추가. 카카오 로그인 후 이름/연락처 자동 채움 → 매장명만 추가 입력
- **Priority**: **High** (단, 개발 리소스 고려 시 중기 과제)

### Issue #10: 모바일 Sticky CTA가 640px 이하에서만 노출
- **Issue**: 태블릿(641~768px)에서는 Sticky CTA가 미노출. 스크롤 깊이 문제가 동일하게 존재
- **Impact**: 낮음. 태블릿 트래픽 비중에 따라 다름
- **Fix**: breakpoint를 768px로 확대
- **Priority**: **Low**

## 3. 추천 변경사항

### Quick Wins (당일 적용 가능)

1. **신뢰 문구 중복 제거** — 폼 하단 2줄을 1줄로 통합
2. **완료 화면 카피 개선** — "24시간 내 카카오톡으로 사용 안내를 보내드려요" 명시
3. **업종 옵션 확장** — 베이커리, 꽃집, 헬스 등 추가
4. **폼 상단 외부 p 태그 제거** — 폼 카드 내부에 메시지 통합

### High-Impact Changes (1주일 이내)

1. **필수 필드 축소** — 이름 + 연락처만 필수, 나머지 선택
2. **인라인 유효성 검사** — 필드별 실시간 에러 표시, alert 제거
3. **연락처 자동 포맷팅** — 하이픈 자동 삽입 + 숫자만 허용
4. **Hero에 간소화 폼 배치** — 또는 Hero 직하단에 폼 이동

### Test Hypotheses (A/B 테스트 대상)

1. **폼 위치 테스트**: 현재 위치(하단) vs Hero 직하단 vs Hero 인라인
2. **필드 수 테스트**: 현재 5개 vs 필수 2개(이름+연락처)만
3. **CTA 카피 테스트**: "무료 테스터 신청하기 →" vs "30초면 끝, 지금 신청 →" vs "남은 자리 확인하기 →"
4. **소셜 프루프 위치 테스트**: Before/After를 폼 바로 위 vs 현재 위치
5. **카카오 간편 신청** vs 수동 폼 전환율 비교

## 4. 최적화된 폼 설계안

### 권장 필드 구성
| 순서 | 필드 | 필수 | 이유 |
|------|------|------|------|
| 1 | 이름 | 필수 | 알림톡 발송 시 호칭 필요 |
| 2 | 연락처 | 필수 | 알림톡 발송 필수 |
| 3 | 매장명 | 선택 | 온보딩 시 수집 가능 |
| 4 | 업종 | 선택 | 온보딩 시 수집 가능 |
| 5 | 인스타그램 계정 | 선택 | 유지 (현재도 선택) |

### 권장 카피
- **폼 타이틀**: "지금 신청하면 무료로 써볼 수 있어요"
- **폼 설명**: "이름과 연락처만 남겨주세요. 24시간 내 카카오톡으로 안내드려요."
- **CTA 버튼**: "무료 테스터 신청하기 →" (현재 유지 — 명확하고 좋음)
- **신뢰 문구**: "카드 등록 없음 · 언제든 탈퇴 가능 · 24시간 내 안내"
- **완료 메시지**: "신청 완료! 24시간 내 카카오톡으로 사용 안내를 보내드려요. 연락처를 확인해 주세요."

```

  [설명: 이름과 연락처만 남겨주세요]
  [슬롯 프로그레스: ██████░░░░ 12/20 신청]
  ──────────────────
  [이름 *]
  [연락처 * (자동 포맷팅)]
  [매장명 (선택)]
  [업종 (선택)]
  [인스타그램 계정 (선택)]

  [무료 테스터 신청하기 → ]
  카드 등록 없음 · 언제든 탈퇴 가능 · 24시간 내 안내
```

## 5. 측정 권장사항

### 핵심 지표 (현재 미측정 추정)
| 지표 | 측정 방법 | 목표 |
|------|----------|------|
| 폼 도달률 | 폼 섹션 IntersectionObserver | > 60% |
| 폼 시작률 | 첫 필드 focus 이벤트 | > 40% (도달자 대비) |
| 폼 완료율 | submit 성공 / 폼 시작 | > 70% |
| 필드별 이탈률 | 각 필드 focus → blur without value | 이탈 필드 식별 |
| CTA 클릭률 | Hero/배너/Sticky CTA별 클릭 추적 | CTA별 비교 |
| 모바일 vs 데스크탑 완료율 | UA 기반 분리 | 격차 < 15% |

### 추적 이벤트 추가 권장
```javascript
// 폼 도달
IntersectionObserver → 'form_view'

// 필드 상호작용
input.focus → 'field_focus', {field: 'name'}
input.blur (empty) → 'field_abandon', {field: 'name'}

// 제출
submit 시작 → 'form_submit_start'
submit 성공 → 'form_submit_success'
submit 실패 → 'form_submit_error', {reason: '...'}

// 공유
shareBeta() → 'post_signup_share', {method: 'kakao' | 'copy'}
```
