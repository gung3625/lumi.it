# 에이전트 작업 보고서 규칙

모든 에이전트는 작업 완료 후 반드시 `docs/agent-reports/{timestamp}-{task-name}.md` 파일에 보고서를 작성해야 합니다.

## 보고서 형식

```markdown
# 에이전트 작업 보고서

- **작업**: {작업 설명}
- **대상 파일**: {수정한 파일 목록}
- **시작 시간**: {시작}
- **완료 시간**: {완료}

## 변경사항
- {변경 1}: {파일명} - {무엇을 어떻게 바꿨는지}
- {변경 2}: ...

## 미반영 항목
- {항목}: {왜 반영하지 않았는지 사유}

## 실패 항목
- {항목}: {실패 원인 설명}

## 비고
- {추가 참고사항}
```

## 규칙
1. 변경사항은 구체적으로 — "헤드라인 변경"이 아니라 "h1 태그의 '인스타 자동 포스팅'을 '사진 찍었는데 글이 안 떠오를 때'로 변경"
2. 미반영 항목은 반드시 사유를 기재 — "이미 다른 에이전트가 반영함", "해당 HTML 요소가 존재하지 않음" 등
3. 실패 항목은 에러 메시지 포함
4. 보고서 파일명: `{YYYYMMDD-HHMMSS}-{task-name}.md` (예: 20260409-091500-page-cro.md)
