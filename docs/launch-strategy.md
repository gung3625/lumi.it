# lumi 베타 런칭 전략

> 작성일: 2026-04-09
> 현재 단계: 프리런칭 베타 (테스터 0명 / 목표 20명)
> 블로커: Meta 앱 재심사 중, Solapi 알림톡 재검수 중

---

## Phase 1: Internal Launch — 내부 검증 (1~2주차)

### 목표
핵심 기능(사진 업로드 → AI 캡션 생성 → 인스타 게시)이 실제로 작동하는지 검증

### 실행 항목

| # | 액션 | 담당 | 완료 기준 |
|---|------|------|----------|
| 1 | 김현님 본인 계정으로 캡션 생성 → 게시 전체 플로우 테스트 | 김현 | 3개 이상 게시 성공 |
| 2 | 업종별 캡션 품질 확인 (카페, 음식점, 뷰티샵 각 1건) | 김현 | 캡션이 업종 맥락에 맞음 |
| 3 | 모바일 웹 플로우 테스트 (사진 촬영 → 업로드 → 게시) | 김현 | 모바일에서 끊김 없이 완료 |
| 4 | Meta 앱 심사 통과 | 김현 | 승인 완료 |
| 5 | Solapi 알림톡 검수 통과 | 김현 | 검수 완료 |
| 6 | 지인 3~5명에게 1:1 데모 + 피드백 수집 | 김현 | 치명적 UX 이슈 0건 |

### 1:1 온보딩 방법
- 카톡/전화로 직접 부탁 (폼 X, 개인적 요청)
- 화면 공유하며 직접 시연 → 어디서 막히는지 관찰
- 피드백 구조화: 캡션 품질(5점), 소요 시간, "꼭 필요/불필요" 기능

### 핵심 질문 (이 단계에서 답해야 할 것)
- 캡션 품질이 "직접 쓴 것보다 낫다"고 느끼는가?
- 사진 업로드부터 게시까지 몇 분 걸리는가? (목표: 2분 이내)
- 해시태그가 실제 도달률에 도움이 되는가?

### Phase 1 완료 기준
지인 3명 이상이 사진→캡션→게시 풀플로우를 막힘 없이 완료

## Phase 2: Alpha Launch — 첫 외부 검증 (3~4주차)

### 목표
모르는 사람이 혼자서 가입→사용할 수 있는지 검증. 초기 대기자 5~10명 확보.

### 실행 항목

| # | 액션 | 상세 | 완료 기준 |
|---|------|------|----------|
| 1 | 랜딩 페이지(lumi.it.kr) 최적화 | "사진만 올리면 AI가 캡션 쓰고 인스타에 올려줌" 메시지 명확히 | 방문자 → 베타 신청 전환율 10%+ |
| 2 | 베타 신청 페이지(lumi.it.kr/beta) 점검 | 신청 폼 동작, 운영자 알림톡, 선착순 카운터 정상 | 신청 → 알림톡 수신 확인 |
| 3 | 소상공인 지인 5~10명에게 개별 초대 | 카톡/문자로 1:1 메시지 발송 | 5명 이상 신청 |
| 4 | 소상공인 커뮤니티 리서치 | 네이버 카페, 당근마켓 동네홍보, 인스타 소상공인 해시태그 | 타겟 커뮤니티 5개 리스트업 |
| 5 | 초기 사용자 온보딩 시나리오 작성 | 가입 → 인스타 연결 → 첫 캡션 생성 → 게시 | 3분 이내 첫 게시 완료 가능 |

### 초대 메시지 템플릿 (카톡용)
```
사장님, 안녕하세요!
인스타 올리기 귀찮으셨죠?

사진만 올리면 AI가 캡션 써주고
인스타에 바로 올려주는 서비스 만들었어요.

지금 무료 테스터 20명 모집 중인데
사장님 피드백 받고 싶어서 연락드려요.

관심 있으시면 여기서 신청해주세요!
https://lumi.it.kr/beta
```

### Phase 2 완료 기준
외부 사용자 5명 이상 셀프 온보딩 성공 + 대기자 10명 이상

## Phase 3: Beta Launch — 20명 확보 + 입소문 (5~8주차)

### 목표
테스터 20명 확보 → 실사용 데이터 수집 → 유료 전환 가능성 검증

### 3-1. 테스터 모집 채널 (ORB 프레임워크)

#### Owned (자체 채널)
| 채널 | 액션 | 우선순위 |
|------|------|---------|
| 랜딩 페이지 | 베타 신청 CTA 상시 노출, 실시간 잔여석 표시 | **최우선** |
| 카카오톡 채널 | 루미 카카오 채널 개설, 신청자에게 진행 상황 공유 | 높음 |
| 블로그 | "소상공인 인스타 자동화" 키워드 SEO 콘텐츠 2~3개 | 중간 |

#### Rented (빌린 채널)
| 채널 | 액션 | 우선순위 |
|------|------|---------|
| 인스타그램 | 루미 공식 계정에서 비포/애프터 캡션 예시 공유 | **최우선** |
| 네이버 카페 | 소상공인 카페 3곳에 후기형 콘텐츠 게시 | 높음 |
| 당근마켓 | 동네 소상공인 대상 홍보글 | 중간 |
| 블라인드/창업 커뮤니티 | 자영업자 게시판에 솔직한 개발 후기 | 중간 |

**커뮤니티 글 톤:** 광고 X → "이런 거 써봤는데 괜찮더라" 후기 형식

#### Borrowed (빌려온 채널)
| 채널 | 액션 | 우선순위 |
|------|------|---------|
| 소상공인 인플루언서 | 카페/음식점 사장님 인스타 계정 3~5곳에 무료 체험 제안 | 높음 |
| 소상공인 유튜버 | "인스타 마케팅 꿀팁" 류 유튜버에게 체험 제안 | 중간 |
| 지역 상권 모임 | 오프라인 사장님 모임에서 데모 | 낮음 (기회 있을 때) |

### 3-2. 테스터 운영 계획

**온보딩 (가입 후 24시간 이내)**
1. 환영 알림톡 발송 (가입 확인 + 시작 가이드 링크)
2. 인스타그램 계정 연결 안내
3. 첫 번째 캡션 생성 직접 도와주기 (필요 시 1:1 카톡)

**피드백 수집 (주 1회)**
- 카톡 1:1 채팅으로 간단 질문 3개:
  1. 이번 주 캡션 몇 개 만드셨나요?
  2. 캡션 품질 만족도 (1~5점)?
  3. 불편한 점이나 바라는 기능?

**핵심 지표 추적**
| 지표 | 목표 | 측정 방법 |
|------|------|----------|
| 테스터 신청 수 | 20명 | Blobs 데이터 |
| 주간 활성 사용률 | 60%+ | 캡션 생성 로그 |
| 캡션 품질 만족도 | 4.0/5.0+ | 주간 피드백 |
| 첫 캡션 생성까지 시간 | 3분 이내 | 온보딩 로그 |
| 유료 전환 의향 | 50%+ | 2주차 설문 |

### 3-3. 희소성 + FOMO 전략

- **"선착순 20명 무료"** — 잔여석 실시간 표시 (예: "남은 자리: 13/20")
- **"베타 테스터 전용 혜택"** — 정식 출시 후 첫 달 50% 할인
- **"테스터 피드백이 제품을 만듭니다"** — 참여 동기 부여
- **추천 인센티브** — "친구 1명 추천하면 1달 추가 무료"
- **마감 임박 알림** — "남은 자리 5명 / 3명 / 마지막 1명"
- 20명 마감 시 → 대기 명단 모드 자동 전환 (이미 구현됨)

### 3-4. 유료 전환 테스트 (Phase 3 후반)

- 테스터 중 NPS 9~10점 사용자에게 먼저 유료 플랜 제안
- "베이직 월 19,000원인데, 테스터 할인으로 첫 3개월 9,900원"
- 전환율 목표: 20명 중 최소 3~5명 유료 전환

### Phase 3 완료 기준
테스터 20명 마감 + 대기자 30명 이상 + 유료 전환 3명 이상

## Pre-Launch 체크리스트 (루미 맞춤)

### 제품 준비
- [ ] Meta 앱 심사 통과 (인스타그램 게시 권한)
- [ ] Solapi 알림톡 검수 통과 (신청 알림)
- [ ] 사진 → 캡션 → 게시 전체 플로우 정상 동작
- [ ] 업종별 캡션 품질 검증 (카페, 음식점, 뷰티샵 최소 3개)
- [ ] 해시태그 자동 생성 정확도 확인 (업종+지역 반영)
- [ ] 모바일 웹 UX 테스트 완료 (소상공인 대부분 모바일 접속)
- [ ] 에러 발생 시 사용자 친화적 메시지 표시

### 랜딩 & 전환
- [ ] 랜딩 페이지 가치 제안 명확 ("사진만 올리면 AI가 캡션 쓰고 인스타에 올려줌")
- [ ] 베타 신청 폼 정상 동작 + 알림톡 수신 확인
- [ ] 선착순 잔여석 실시간 표시
- [ ] 신청 완료 화면/메시지 준비 ("24시간 내 안내 드릴게요")
- [ ] 모바일 최적화 확인

### 마케팅 자산
- [ ] 스크린샷 3장 이상 (캡션 생성 화면, 게시 완료 화면, 대시보드)
- [ ] 사용 과정 GIF 또는 15초 영상
- [ ] 업종별 캡션 샘플 (카페/음식점/뷰티샵 각 2개)
- [ ] 비포/애프터 캡션 비교 콘텐츠
- [ ] "사진만 올리면 끝" 핵심 메시지 통일

### 채널 준비
- [ ] 루미 인스타그램 공식 계정 개설 + 프로필 최적화
- [ ] 카카오톡 채널 개설 (선택)
- [ ] 소상공인 타겟 커뮤니티 5곳 리스트업 + 가입
- [ ] 초대 메시지 템플릿 준비 (카톡, DM용)
- [ ] 지인 네트워크 중 소상공인 리스트업 (Phase 1 대상)

### 온보딩 & 피드백
- [ ] 신규 테스터 온보딩 시나리오 문서화
- [ ] 주간 피드백 수집 프로세스 확정
- [ ] 테스터 관리 스프레드시트 (이름, 업종, 신청일, 초대일, 상태)
- [ ] 피드백 수집 템플릿 (캡션 품질, 사용성, 개선 요청)

### 분석 & 추적
- [ ] 베타 신청 수 실시간 추적 (Blobs)
- [ ] 신청 전환율 추적 (랜딩 방문 → 베타 신청)
- [ ] 온보딩 완료율 추적 (초대 → 첫 캡션 생성)
- [ ] 캡션 사용률 추적 (생성된 캡션 중 실제 게시 비율)
- [ ] NPS 측정 준비 (테스터 10명 도달 시 첫 측정)
- [ ] 에러 로그 모니터링

### 운영 준비
- [ ] 장애 대응 프로세스 (Meta API 에러, 캡션 생성 실패 시 대응)
- [ ] 20명 마감 시 대기 명단 자동 전환 정상 동작 확인

## 타임라인 요약

```
[1~2주차] Phase 1: 내부 검증
  └─ 본인 테스트 + 지인 데모 + 블로커(Meta/Solapi) 해소

[3~4주차] Phase 2: 첫 외부 사용자
  └─ 지인 5~10명 초대 + 랜딩 최적화 + 커뮤니티 리서치

[5~8주차] Phase 3: 20명 확보
  └─ 커뮤니티 홍보 + 인플루언서 체험 + 피드백 루프 + 유료 전환 테스트

[9주차~] Phase 4 준비
  └─ 대기자 배치 초대 + 사용 데이터 기반 제품 개선
```

## 핵심 원칙

1. **한 명씩, 직접** — 초기에는 스케일보다 깊이. 1:1 온보딩이 최우선
2. **후기가 마케팅** — 테스터의 실제 사용 후기 > 우리가 만든 광고 카피
3. **희소성 활용** — "선착순 20명"은 진짜 마감. 대기 명단으로 FOMO 극대화
4. **블로커 먼저** — Meta 심사/Solapi 검수 해결 전까지 마케팅 노출 자제
5. **유료 전환이 진짜 검증** — 무료로 쓰는 건 검증이 아님. 돈을 내야 PMF

## 다음 단계 (Phase 3 완료 후)

1. **Phase 4 (Early Access)**: 대기자 명단에서 배치 초대 (5~10명씩)
2. **Phase 5 (Full Launch)**: 자가 가입 오픈 + 전 채널 풀 캠페인 + 정가 과금
3. **Product Hunt 런칭**: Phase 5 시점에 맞춰 리스팅 준비
