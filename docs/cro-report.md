## 1. 랜딩페이지(index.html) 전환율 분석

### 1-1. 현황 요약

| 항목 | 현재 상태 | 판정 |
|------|----------|------|
| 5초 내 가치 전달 | 히어로 "우리 매장을 아는 lumi가 대신 써드려요" — 무엇을 대신 써주는지 즉시 이해 어려움 | 개선 필요 |
| 주요 CTA | "무료 테스터 신청" — /beta로 이동 | 적절 |
| 보조 CTA | "직접 체험하기" — 데모 섹션 스크롤 | 좋음 |
| Trust Signal | 후기 4건 + Before/After 비교 | 보통 |
| 페이지 길이 | 매우 김 (히어로→메트릭→사용법→기능→데모→비교→후기→가격→CTA→FAQ→Footer) | 과다 |
| 모바일 Sticky CTA | 있음 (768px 이하) | 좋음 |

### 1-2. Value Proposition Clarity (가치 제안 명확성)

**문제점:**
- 히어로 헤드라인 "우리 매장을 아는 lumi가 대신 써드려요"가 **무엇을** 대신 써주는지 불명확
- 서브카피에서야 "캡션 · 해시태그 · 게시까지 전부 자동"이 나옴 → 핵심이 묻힘
- "우리 매장을 아는"이라는 수식어가 첫 방문자에게 와닿지 않음 (아직 써보지 않아서)

**개선안:**
- 헤드라인에 **구체적 결과물**(캡션, 인스타 게시)을 직접 언급
- "아는"이라는 추상적 표현 → 구체적 증거(날씨, 말투)로 교체

### 1-3. CTA 분석

**문제점:**
- Primary CTA "무료 테스터 신청"이 /beta 페이지로 이동 → 전환 단계 추가 (마찰)
- 데모 체험 후 CTA가 너무 많음 (업그레이드 카드 + 비교표 + 톤 CTA + 별도 CTA 박스)
- 가격 섹션의 CTA가 모두 "/beta"로 통일 → 현재 단계에 적합하지만, 차별화 없음

**개선안:**
- 히어로 CTA에 긴급성 추가: "무료 테스터 신청 (14/20 마감)"
- 데모 후 CTA를 하나로 통합 → 분산된 3개 CTA가 혼란 유발

### 1-4. Social Proof (사회적 증거)

**문제점:**
- 메트릭 "4.9점 테스터 평균 만족도" — 현재 테스터 0명인데 4.9점은 신뢰도 훼손
- 후기 4건 모두 ★★★★★ / ★★★★☆ — 베타 전인데 후기가 있으면 의심
- 후기 인물(김민정, 박서준, 이지현, 최지우)이 실제인지 확인 불가

**개선안 (단기):**
- 메트릭 섹션의 "4.9점"을 삭제하거나, "내부 테스트" 명시
- 후기를 "파일럿 테스트 참여자" 등으로 명확히 표기
- 또는 후기 섹션을 아예 제거하고, 대신 Before/After 비교만 강화

**개선안 (중기):**
- 실제 테스터 확보 후 리얼 후기로 교체
- 사진 + 매장 인스타 링크 포함 → 검증 가능한 후기

### 1-5. 마찰 포인트 (Friction)

| 위치 | 마찰 | 심각도 |
|------|------|--------|
| 히어로 → 베타 페이지 | 2단계 전환 (클릭 → 새 페이지 → 폼 작성) | 중 |
| 데모 섹션 | 결과 후 CTA가 3곳에 분산 | 중 |
| 기능 섹션 | "모든 기능 보기" 버튼 → 추가 9개 카드 → 정보 과부하 | 낮 |
| 가격 → 구독 | /subscribe로 이동 → 로그인 필요 → 결제 | 높 |

## 2. 히어로 섹션 & CTA 카피 개선안

### 2-1. 현재 히어로 카피

```
칩: 베타 테스터 모집 중
헤드라인: 우리 매장을 아는 lumi가 대신 써드려요.
서브카피: 날씨, 동네 행사, 우리 매장 말투까지 이해하고
         사진 한 장으로 캡션 · 해시태그 · 게시까지 전부 자동.
CTA: 무료 테스터 신청 / 직접 체험하기
```

### 2-2. 헤드라인 대안

**Option A (결과 중심 — 추천):**
```
사진 한 장이면
인스타 게시까지 끝.
```
- 근거: 핵심 가치를 5초 내 전달. "사진 한 장"이라는 최소 입력 + "게시까지 끝"이라는 최종 결과
- 서브카피: "캡션, 해시태그, 예약 게시까지 lumi가 전부 해드려요. 매일 30분 쓰던 인스타, 1분이면 끝."

**Option B (Pain point 직격):**
```
인스타 글 쓰느라
가게 문 늦게 열지 마세요.
```
- 근거: 소상공인의 실제 페인포인트 자극 → 감정적 공감
- 서브카피: "사진만 올리면 lumi가 캡션 써서 인스타에 올려드려요. 날씨, 행사, 말투까지 알아서."

**Option C (수치 기반):**
```
인스타 30분 → 47초.
사진만 올리면 됩니다.
```
- 근거: 구체적 숫자가 신뢰감 형성. 현재 메트릭의 "47초"를 헤드라인으로 끌어올림
- 서브카피: "AI가 사진 보고 캡션 써서 인스타에 올려드려요. 해시태그, 예약 게시까지 전부 자동."

### 2-3. CTA 카피 대안

| 현재 | 개선안 | 근거 |
|------|--------|------|
| 무료 테스터 신청 | 무료로 시작하기 | "신청"은 심사 느낌 → "시작"은 즉시성 |
| 무료 테스터 신청 | 지금 무료로 써보기 | 행동 중심 + 가치(무료) 강조 |
| 직접 체험하기 | 30초 데모 해보기 | 시간 명시 → 부담 감소 |

### 2-4. 서브 CTA (가격 섹션 하단)

현재: "시작하기" / "무료 테스터 신청하기"
개선안: 베이직 "월 19,000원으로 시작하기" / 스탠다드 "가장 인기 — 무료로 먼저 써보기"

## 3. 전체 사이트 카피 리뷰 (Seven Sweeps)

### Sweep 1: Clarity (명확성)

| 위치 | 문제 | 개선안 |
|------|------|--------|
| 히어로 "대신 써드려요" | 무엇을 써주는지 불명확 | "캡션 써서 인스타에 올려드려요" |
| 메트릭 "준비물 전부" | "사진 1장"이 준비물이라는 연결이 약함 | "필요한 건 사진 1장뿐" |
| 기능 "글 고민 0분" | 카드 제목이 기능이 아닌 결과 — 좋지만, 바로 아래 설명이 같은 말 반복 | 설명에 구체적 시나리오 추가 |
| 데모 "lumi가 바로 캡션을 써드릴게요" | "써드릴게요"가 어색 (존댓말 혼용) | "캡션을 만들어 드려요" |

### Sweep 2: Voice & Tone (톤 일관성)

- **전반적으로 양호**: 친근 + 실용적 톤 유지
- **문제점**: 일부 섹션에서 톤 불일치
  - 히어로: "대신 써드려요" (자연체)
  - 메트릭: "평균 47초", "3가지" (데이터 톤)
  - 기능 카드: "도달률 높이는 태그" (마케팅 전문 용어)
  - FAQ: "아니요, 비밀번호는 절대 필요하지 않아요" (설명체)
- **개선안**: "도달률" → "검색에 잘 걸리는 태그" (사장님 언어로)

### Sweep 3: So What (그래서 뭐?)

| 현재 카피 | So What 테스트 | 개선안 |
|----------|---------------|--------|
| "해시태그 자동 추천" | 그래서? → 검색 노출 증가 | "검색에 잘 걸리는 태그를 알아서 달아줘요" |
| "예약 게시" | 그래서? → 시간 자유 | "원하는 시간에 자동으로 올라가요. 새벽에 안 깨도 돼요" |
| "7일 콘텐츠 캘린더" | 그래서? → 촬영 고민 해결 | "'오늘 뭐 찍지?' 고민이 사라져요" (현재 설명에 이미 있음 ✓) |
| "1:1 말투 학습" | 그래서? → 점점 우리 매장답게 | "쓸수록 우리 매장 톤으로 맞춰져요" |

### Sweep 4: Prove It (증거)

**미증명 주장:**
- "평균 47초" — 출처 불명, 테스터 0명인데 어디서 나온 수치?
- "4.9점 테스터 평균 만족도" — 실제 테스터 없음
- "좋아요 128개" (폰 목업 내) — 허구적 수치

**조치:**
- 47초 → 내부 테스트 결과임을 명시하거나, 실제 데이터 확보 후 교체
- 4.9점 → 삭제하거나 "내부 파일럿" 표기
- 후기 → 파일럿 테스터 표기 추가

### Sweep 5: Specificity (구체성)

| 모호 | 구체적 대안 |
|------|-----------|
| "시간 절약" | "매일 30분 → 1분" (현재 OG태그에 37분으로 되어 있으나 본문과 불일치) |
| "정식 출시 특별 혜택" | "정식 출시 시 첫 3개월 50% 할인" (구체적 숫자) |
| "자동으로 올라가요" | "설정한 시간에 자동으로 인스타에 게시돼요" |

### Sweep 6: Heightened Emotion (감정)

**현재 잘 되어 있는 부분:**
- Before/After 비교가 효과적 (직접 쓴 밋밋한 글 vs lumi의 자연스러운 글)
- "딸기 라떼 올리려고 사진 찍다가 크림 흘림 (3번째)" — 리얼한 톤

**강화 가능:**
- 히어로에 페인포인트 한 줄 추가: "매일 밤 인스타 글 쓰느라 지치셨죠?"
- FAQ 전에 공감 섹션: "사장님도 이런 고민 있으시죠?" + 체크리스트

### Sweep 7: Zero Risk (리스크 제거)

| 위치 | 현재 | 개선안 |
|------|------|--------|
| 히어로 근처 | 없음 | "카드 등록 없이 무료로 시작" 마이크로카피 추가 |
| 데모 CTA | 없음 | "30초면 끝나요" 추가 |
| 가격표 하단 | 없음 | "7일 무료 체험 · 언제든 해지" 추가 |
| 베타 폼 근처 | "신용카드 불필요 · 스팸 없음" | ✓ 잘 되어 있음 |

## 4. 베타 신청 흐름(beta.html) 최적화

### 4-1. 현재 흐름

```
배너(상단) → 히어로 → HOW → 혜택 → 기능 11개 → Before/After → 폼(5필드) → FAQ → Footer
```

### 4-2. 폼 필드 분석

| 필드 | 필수 여부 | 판정 |
|------|----------|------|
| 이름 | 필수 | 필요 (알림톡 발송용) |
| 매장명 | 필수 | 필요 (캡션 커스텀용) |
| 업종 | 필수 | 필요 (업종별 캡션 생성용) |
| 연락처 | 필수 | 필요 (알림톡 발송용) |
| 인스타그램 계정 | 선택 | 나중에 수집 가능 → 제거 권장 |

### 4-3. 문제점 및 개선안

**문제 1: 페이지가 너무 김**
- 베타 신청 페이지인데 기능 소개가 11개 카드 → 정보 과부하
- 사용자가 폼까지 스크롤하는 데 너무 오래 걸림
- **개선안**: 기능 섹션을 3~4개로 축소하거나, 히어로 바로 아래에 폼 배치

**문제 2: 폼 위치가 페이지 하단**
- 스크롤 50% 이상 해야 폼에 도달
- 모바일 Sticky CTA가 있지만, 데스크톱에서는 없음
- **개선안**: 
  - 히어로 → 3단계 HOW → 폼 (간소화 배치)
  - 또는 히어로에 인라인 폼 임베드

**문제 3: 폼 하단 중복 안내**
- "언제든 탈퇴 가능 · 카드 등록 없음 · 24시간 내 안내"
- "신용카드 불필요 · 스팸 없음 · 언제든 취소 가능"
- 같은 내용이 2줄 → 하나로 통합

**문제 4: 슬롯 바 효과**
- 20명 중 n명 신청 완료 시각화가 있음 — 좋은 긴급성 장치
- 하지만 현재 0명이면 긴급성이 작동하지 않음
- **개선안**: "4월 마감 예정"을 더 강조. 슬롯이 0일 때는 "지금 신청하면 첫 번째 테스터"로 표현

**문제 5: 인스타 계정 필드**
- "(선택)"이라고 되어 있지만 필드가 있으면 마찰 증가
- **개선안**: 제거 후, 가입 완료 → 대시보드에서 연동 유도

### 4-4. 최적화된 폼 구조 제안

```
[폼 카드]
  배지: 🟢 무료 테스터 모집 중
  제목: 무료로 먼저 써보세요
  설명: 선착순 20명 · 정식 출시 전까지 모든 기능 무료
  
  [이름] ← placeholder: "홍길동"
  [매장명] ← placeholder: "성수 카페 루미"  
  [업종] ← 드롭다운
  [연락처] ← placeholder: "010-0000-0000"
  
  [무료 테스터 신청하기 →] 버튼
  
  마이크로카피: "카드 등록 없음 · 24시간 내 안내 · 언제든 탈퇴 가능"
```

## 5. 이탈 방지 팝업 제안

### 5-1. 현재 상태

현재 lumi 사이트에 이탈 방지 팝업 없음.

### 5-2. 제안: Exit Intent 팝업 (데스크톱)

**트리거:** 마우스가 브라우저 상단으로 이동 (exit intent)
**빈도:** 세션당 1회, 닫으면 7일간 미표시 (localStorage)
**대상:** 미가입 방문자만 (localStorage에 lumi_token 없는 경우)

**카피:**
```
헤드라인: 잠깐, 사진 한 장만 올려보세요.
서브카피: 체험판으로 캡션 하나 받아가세요. 
         가입 없이, 지금 바로.

[CTA] 무료 캡션 받아보기 →
[닫기] 괜찮아요, 다음에 올게요
```

**디자인:**
- 중앙 모달, 400px 폭
- 다크 배경 (#272729), 핑크 CTA
- 닫기 버튼(✕) 우상단 명확히
- 클릭 시 #demo 섹션으로 스크롤

### 5-3. 제안: 스크롤 기반 Slide-in (모바일 대안)

**트리거:** 50% 스크롤 도달 + 30초 체류
**위치:** 하단 우측 코너 slide-in
**빈도:** 세션당 1회

**카피:**
```
🟢 지금 신청하면 14번째 테스터
[무료 신청 →]
```

### 5-4. 제안: 데모 완료 후 인라인 CTA 강화

현재 데모 결과 후 CTA가 3곳에 분산 → 1곳으로 집중:

```
[데모 결과 카드]
  ↓
[단일 CTA 카드]
  "이 캡션은 사진만 보고 만든 거예요.
   가입하면 날씨, 말투, 트렌드까지 반영해서 써드려요."
  [무료로 시작하기 →]
```

## 6. 가격 정책 분석 + 개선안

### 6-1. 현재 가격 구조

| 플랜 | 가격 | 가치 메트릭 | 핵심 차별점 |
|------|------|-----------|-----------|
| 베이직 | ₩19,000/월 | 캡션 주 3개 | 바로 게시만 |
| 스탠다드 | ₩29,000/월 | 무제한 캡션 | 예약게시 + 날씨 + 말투학습 |
| 프로 | ₩39,000/월 | 무제한 캡션 | 캘린더 + 트렌드 + 최적시간 |

### 6-2. 분석

**잘 된 점:**
- 3-tier Good-Better-Best 구조 적절
- 추천 플랜(스탠다드) 명확히 표시
- 가격 간격(₩10,000)이 균등 → 스탠다드가 자연스런 선택
- "자동 갱신 없음" → 한국 소상공인 정서에 맞는 리스크 제거

**문제점:**

1. **베이직이 너무 제한적**
   - 주 3개 캡션 = 월 12개 → 소상공인이 매일 올리려면 부족
   - 예약 게시, 날씨, 말투 학습 모두 없음 → "쓸 이유가 없는" 플랜
   - 베이직 구매자가 불만 → 이탈 가능성

2. **스탠다드 vs 프로 차별화 약함**
   - 프로에만 있는 "캘린더, 트렌드, 최적 시간" — 소상공인이 얼마나 가치를 느낄지 불확실
   - ₩10,000 차이에 비해 체감 가치가 낮을 수 있음

3. **무료 체험 기간 없음**
   - "무료 테스터"가 체험 역할이지만, 정식 출시 후에는?
   - 7일 또는 14일 무료 체험이 전환율을 크게 높일 수 있음

4. **연간 구독 옵션 없음**
   - 월 결제만 → ARPU 최적화 기회 놓침
   - 연간 구독 시 2개월 무료(17% 할인) 제안 가능

### 6-3. 가격 개선안

**단기 (베타 기간):**
- 현재 가격 유지, 테스터 피드백으로 WTP(지불의향) 데이터 수집
- 테스터에게 "이 기능에 얼마까지 낼 수 있나?" 질문 (Van Westendorp)

**중기 (정식 출시):**

| 플랜 | 현재 | 제안 | 변경 이유 |
|------|------|------|----------|
| 베이직 | ₩19,000 | ₩15,000 | 진입 장벽 낮추기. 주 3개 → 주 5개로 상향 |
| 스탠다드 | ₩29,000 | ₩29,000 | 유지. 앵커 역할 |
| 프로 | ₩39,000 | ₩49,000 | 가격 올리고 프리미엄 기능 추가 (리포트, 경쟁사 분석) |

**추가 제안:**
- 연간 구독 도입: 월 가격의 10개월분 (2개월 할인)
- 베이직 → 스탠다드 업그레이드 시 첫 달 ₩10,000 프로모션

### 6-4. 가격 페이지 UX 개선

| 항목 | 현재 | 개선안 |
|------|------|--------|
| 월/연 토글 | 없음 | 연간 구독 추가 시 토글 필요 |
| "누구를 위한 플랜" | "인스타 시작하는 매장" 등 한 줄 | 각 플랜 하단에 추천 대상 명시 |
| 비교표 | 없음 (각 카드에 개별 리스트) | 가격 섹션 하단에 전체 비교표 추가 |
| FAQ | 가격 관련 FAQ 없음 | "중간에 플랜 변경 가능한가요?" 등 추가 |
