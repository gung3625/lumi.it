# lumi 홈페이지 카피라이팅 분석 및 개선안

> 분석 기준: Copywriting Principles (Clarity Over Cleverness, Benefits Over Features, Specificity Over Vagueness, Customer Language) + Page Structure Framework (Above the Fold, Core Sections)
>
> 분석 일자: 2026-04-09

---

## 1. 현재 카피 진단 요약

### 잘 되어 있는 점
- **고객 언어 사용**: "사장님", "우리 매장", "사진 찍다가 크림 흘림" 등 타겟이 쓰는 말투 활용
- **Before/After 비교**: 직접 쓴 글 vs lumi 글 대비가 설득력 있음
- **체험형 데모**: 사진 올려 직접 캡션 생성 체험 — 강력한 전환 장치
- **구체적 숫자**: "37분 → 1분", "주 3개", "최대 10장" 등

### 개선이 필요한 점

| 원칙 | 현재 문제 | 위치 |
|------|----------|------|
| **Clarity Over Cleverness** | 헤드라인 "우리 매장을 아는 lumi가 대신 써드려요" — "대신 써드려요"가 무엇을(캡션? 글?) 명확하지 않음 | Hero H1 |
| **Benefits Over Features** | 기능 섹션이 Feature 나열 위주. "글 고민 0분", "도달률 높이는 태그"는 좋으나 "여러 장도 한 번에", "스토리 자동 게시"는 기능 설명에 그침 | Features 섹션 |
| **Specificity Over Vagueness** | "검색 노출이 확 달라져요", "조회수가 다르더라고요" — 구체적 수치 없이 모호 | Features, 후기 |
| **Customer Language** | "도달률"은 마케터 용어. 사장님들은 "사람들이 더 많이 보는", "검색에 더 잘 뜨는"으로 표현 | Features |
| **One Idea Per Section** | Hero에 칩(베타 모집) + 헤드라인(가치제안) + 메트릭 3개 + 모형 — 포커스 분산 | Hero |
| **CTA 약함** | "무료 테스터 신청" 반복 7회. 동일 CTA 반복은 클릭 피로 유발. 맥락별 CTA 변형 필요 | 전체 |

## 2. 헤드라인 대안 3개

### 현재
> 우리 매장을 아는 lumi가 대신 써드려요.

### 대안 A — Pain Point 직격 (추천)
> **사진 찍었는데 글이 안 떠오를 때,**
> **lumi가 1분 안에 써드려요.**

- 원칙: Specificity ("1분 안에") + Customer Language ("글이 안 떠오를 때")
- 근거: 타겟의 가장 큰 고통은 "사진은 찍었는데 뭐라고 쓰지?"

### 대안 B — 결과 중심
> **사진 한 장이면**
> **오늘 인스타 끝.**

- 원칙: Clarity + Benefit (시간 절약이라는 결과)
- 근거: 기존 OG 타이틀("사진 한 장이면 인스타 끝")이 이미 강력 — 본문 헤드라인으로 승격

### 대안 C — 질문형 (공감 유도)
> **매일 인스타 글 쓰느라**
> **30분씩 쓰고 있진 않으세요?**

- 원칙: Customer Language + Rhetorical Question
- 근거: 질문이 독자를 멈추게 하고, 서브헤드라인에서 해결책 제시

## 3. CTA 대안 3개

### 현재
> 무료 테스터 신청 / 무료 테스터 신청하기

### 대안 A — 행동 + 결과 (추천)
> **1분 만에 캡션 받아보기**

- 원칙: [Action Verb] + [What They Get]. "신청"보다 "받아보기"가 즉시 가치를 전달
- 사용 위치: Hero CTA, 최종 CTA

### 대안 B — 리스크 제거 강조
> **무료로 시작하기 — 카드 등록 없음**

- 원칙: Risk Reversal. 가장 큰 저항(돈, 카드)을 CTA 안에서 해소
- 사용 위치: Pricing 섹션 CTA

### 대안 C — 긴급성 + 희소성
> **남은 자리 확인하고 신청하기**

- 원칙: Specificity + Urgency. 잔여 숫자를 동적으로 보여주면서 클릭 유도
- 사용 위치: 배너, 최종 CTA

## 4. 섹션별 개선 카피

### 4-1. Above the Fold (Hero)

| 요소 | 현재 | 개선안 |
|------|------|--------|
| **Chip** | 베타 테스터 모집 중 | 선착순 20명 무료 체험 중 |
| **H1** | 우리 매장을 아는 lumi가 대신 써드려요. | **사진 찍었는데 글이 안 떠오를 때, lumi가 1분 안에 써드려요.** |
| **Sub** | 날씨, 동네 행사, 우리 매장 말투까지 이해하고 사진 한 장으로 캡션·해시태그·게시까지 전부 자동. | **사진만 올리면 우리 매장 분위기에 맞는 캡션이 완성되고, 해시태그까지 달아서 인스타에 바로 올라가요.** |
| **CTA 1** | 무료 테스터 신청 | **1분 만에 캡션 받아보기** |
| **Metric** | 사진 1장 / 준비물 전부 | 사진 1장 → 캡션 완성까지 **58초** |
| **Metric** | 4.9점 / 테스터 평균 만족도 | 테스터 만족도 **4.9**/5.0 |
| **Metric** | 3가지 / 매번 고를 수 있는 캡션 | 매번 **3개 캡션** 중 선택 |

**개선 근거:**
- 서브헤드라인: "날씨, 동네 행사"는 부가 기능이지 핵심이 아님. 첫 화면에서는 코어 밸류(사진→캡션→게시)에 집중
- 메트릭: "준비물 전부"보다 "58초"가 더 구체적이고 인상적

### 4-2. How It Works (사진 한 장이면 이 모든 게 자동이에요)

| 요소 | 현재 | 개선안 |
|------|------|--------|
| **섹션 타이틀** | 사진 한 장이면 이 모든 게 자동이에요 | **3단계면 오늘 인스타 끝이에요** |
| **STEP 01** | 사진 올리기 | **매장 사진 올리기** — 오늘 찍은 사진 한 장이면 충분해요 |
| **STEP 02** | lumi 사진 분석 | **lumi가 사진 읽는 중** — 메뉴, 분위기, 계절감까지 파악해요 |
| **STEP 03** | 캡션 자동 작성 | **캡션 3개 완성** — 마음에 드는 걸 고르세요 |
| **STEP 04** | (제목 없음, 게시 화면) | **인스타에 바로 업로드** — 복사·붙여넣기 없이 자동 게시 |

**개선 근거:**
- "이 모든 게"가 모호 → "3단계"로 구체화하면 간단함이 즉시 전달
- 각 스텝에 서브 설명 추가로 불안감 해소 (Reduce Perceived Complexity)

---

### 4-3. Features (우리 매장 상황을 읽고 알아서 써드려요)

| 요소 | 현재 | 개선안 |
|------|------|--------|
| **섹션 라벨** | 자동화가 아니라 이해 | 다른 AI랑 뭐가 다르냐면 |
| **섹션 타이틀** | 우리 매장 상황을 읽고 알아서 써드려요 | **동네 날씨, 근처 축제, 사장님 말투까지 반영해요** |
| **글 고민 0분** | 사진만 올리면 우리 매장 분위기에 맞는 캡션이 바로 완성돼요. 글 쓰느라 30분씩 고민할 필요 없어요. | **매일 30분씩 글 쓰던 시간, 이제 사진 한 장이면 끝.** 캡션이 바로 나오니까 촬영에만 집중하세요. |
| **도달률 높이는 태그** | 지금 사람들이 검색하는 해시태그를 실시간 분석해서 달아줘요. 검색 노출이 확 달라져요. | **요즘 사람들이 검색하는 태그를 골라 달아줘요.** 같은 사진이어도 태그 하나로 노출이 2~3배 차이나요. |
| **오늘 날씨가 곧 콘텐츠** | 비 오는 날엔 비 감성, 맑은 날엔 맑은 감성. 날씨에 맞는 글이 공감을 끌어요. | **비 오는 날 빵 사진 올리면 "이런 날엔 따뜻한 빵이죠" 같은 글이 나와요.** 날씨가 공감 포인트가 돼요. |
| **최적 시간에 자동 게시** | 팔로워가 가장 활발한 시간에 자동으로 올라가요. 퇴근 후 새벽에 올릴 필요 없어요. | **밤 11시에 올릴 필요 없어요.** 우리 팔로워가 가장 많이 보는 시간에 자동으로 올라가요. |
| **우리 매장 말투로만** | 따뜻한 감성 카페면 따뜻하게, 트렌디한 뷰티샵이면 트렌디하게. 말투·업종·지역을 학습해서 써요. | **감성 카페는 감성으로, 힙한 헤어샵은 힙하게.** 피드백 줄수록 사장님 말투에 가까워져요. |
| **여러 장도 한 번에** | 사진 여러 장을 한꺼번에 올리면 묶음 게시물로 올라가요. 최대 10장까지. | **사진 10장까지 한 번에 올리면 캐러셀 게시물로 완성.** 하나하나 따로 올릴 필요 없어요. |

**개선 근거:**
- "도달률" → "노출" (고객 언어)
- 추상적 설명("공감을 끌어요") → 구체적 예시("이런 날엔 따뜻한 빵이죠")
- "밤 11시에 올릴 필요 없어요" — 사장님의 실제 고충을 먼저 언급 (Pain → Solution)

### 4-4. Demo (직접 체험해 보세요)

| 요소 | 현재 | 개선안 |
|------|------|--------|
| **섹션 라벨** | 직접 체험해 보세요 | 회원가입 없이 바로 체험 |
| **섹션 타이틀** | 사진 한 장 올려보세요, lumi가 바로 캡션을 써드릴게요 | **지금 매장 사진 한 장 올려보세요** |
| **CTA 버튼** | 캡션 만들어 보기 | **내 캡션 받아보기** |
| **업그레이드 라벨** | 지금 이 캡션은 사진만 보고 만들었어요 | **방금 본 캡션, 이게 끝이 아니에요** |
| **업그레이드 CTA** | 무료 테스터로 시작하기 | **더 정교한 캡션 받아보기** |

**개선 근거:**
- "회원가입 없이" 명시 → 진입 저항 감소
- "내 캡션" — 소유감 부여 (Endowment Effect)
- 데모 후 업그레이드 CTA가 "무료 테스터"보다 "더 정교한 캡션"이 자연스러운 흐름

### 4-5. Proof / Social Proof (사장님이 쓴 글 vs lumi가 쓴 글)

| 요소 | 현재 | 개선안 |
|------|------|--------|
| **섹션 라벨** | 같은 사진, 다른 글 | 같은 사진인데 반응이 다릅니다 |
| **섹션 타이틀** | 사장님이 쓴 글 vs lumi가 쓴 글 | **30초 만에 이렇게 달라져요** |
| **Before 라벨** | Before — 직접 쓴 글 | 사장님이 5분 고민해서 쓴 글 |
| **After 라벨** | After — lumi가 쓴 글 | lumi가 30초 만에 쓴 글 |

**개선 근거:**
- 시간 대비 명시("5분 vs 30초")로 구체성 확보
- "반응이 다릅니다"로 결과(Benefit) 중심 프레이밍

### 4-6. Pricing (우리 매장에 맞는 플랜을 선택하세요)

| 요소 | 현재 | 개선안 |
|------|------|--------|
| **섹션 타이틀** | 우리 매장에 맞는 플랜을 선택하세요 | **대행사 월 50만원 vs lumi 월 1.9만원** |
| **베이직 서브** | 인스타 시작하는 매장 | 일주일에 2~3번 올리는 매장 |
| **스탠다드 서브** | 꾸준히 운영하는 매장 | 매일 올리고 싶은 매장 |
| **프로 서브** | 자동화 풀옵션 | 인스타를 매출 채널로 키우는 매장 |
| **스탠다드 CTA** | 무료 테스터 신청하기 | **20명 한정 무료 체험 시작** |

**개선 근거:**
- 타이틀에 가격 앵커링 ("대행사 50만원 vs 1.9만원") — 가격 저항 즉시 해소
- 플랜 서브카피를 행동 기반으로 변경 → "나는 어디에 해당하지?" 판단 용이
- "풀옵션"은 자동차 용어 — "매출 채널로 키우는"이 목표 중심

### 4-7. Final CTA (사장님, 오늘부터 인스타 걱정 끝내세요)

| 요소 | 현재 | 개선안 |
|------|------|--------|
| **H2** | 사장님, 오늘부터 인스타 걱정 끝내세요. | **내일 올릴 인스타, 오늘 1분이면 끝나요.** |
| **서브텍스트** | 잔여 6자리 · 정식 출시 전까지 전부 무료 · 카드 등록 없음 | 선착순 20명 중 **N자리 남음** · 베타 기간 완전 무료 · 카드 등록 없음 |
| **CTA** | 무료 테스터 신청하기 | **남은 자리 확인하고 시작하기** |

**개선 근거:**
- "인스타 걱정"은 추상적 → "내일 올릴 인스타"가 구체적 상황
- CTA에 호기심 요소("남은 자리 확인") 추가로 클릭 유도
- "잔여 6자리"보다 "N자리 남음"이 더 직관적

### 4-8. FAQ

| 현재 질문 | 개선안 |
|----------|--------|
| 베타 기간이 끝나면 바로 유료인가요? | 무료 체험 끝나면 자동 결제되나요? |
| lumi가 쓴 글이 어색하지 않나요? | AI가 쓴 티 안 나나요? |

| 하루에 몇 개까지 올릴 수 있나요? | (유지 — 이미 좋음) |

**개선 근거:**
- FAQ 질문은 고객이 실제로 하는 말투로 (Customer Language)
- "어색하지 않나요"보다 "AI가 쓴 티 안 나나요"가 실제 걱정 표현

### 4-9. Meta / SEO

| 요소 | 현재 | 개선안 |
|------|------|--------|
| **title** | lumi — 홍보, 이제 사진 한 장으로 끝내요 | lumi — 사진 한 장이면 인스타 캡션·게시 자동 |
| **description** | 사진 한 장만 올리면, 나머지는 lumi가 다 해요. | 매장 사진 올리면 AI가 캡션·해시태그 써서 인스타에 바로 올려줘요. 월 1.9만원, 지금 무료 체험 중. |
| **og:title** | lumi — 사진 한 장이면 인스타 끝 | (유지 — 이미 강력) |
| **og:description** | 캡션·해시태그·예약 게시까지 전부 자동. 매일 37분 → 1분. 지금 무료 테스터 모집 중 | 매장 사진 올리면 캡션·해시태그·게시까지 자동. 대행사 비용의 1/25. 선착순 무료 체험 중. |

**개선 근거:**
- title에 핵심 키워드("인스타 캡션", "자동") 포함 → SEO
- description에 가격 앵커("월 1.9만원") 추가 → CTR 향상
- "37분 → 1분"은 출처 불명확한 수치 → "대행사 비용의 1/25"가 검증 가능

## 5. 우선순위 실행 순서
